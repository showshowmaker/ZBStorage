使能分布式存储架构及关键技术V2.0；《面向新型计算模式的 AI 使能分布式存储架构及关键技术测试大纲-V2.0》。系统概述面向新型计算模式的 AI 使能分布式存储架构如图1.2.2-1所示。AI使能的分布式存储控制面和数据面技术包括4个关键模块：用户负载分析与预测技术、系统性能自动调优技术、冷热数据感知技术和智能缓存技术，如图1.2.2-2所示。其中：1. 用户负载分析与预测技术，通过分析用户负载特征，构建用户负载、多维度资源和系统性能的关系模型，完成对未来一段时间内多尺度、多维度的用户负载预测； 2. 系统性能自动调优技术，针对系统参数进行重要性排序，筛选影响力高的参数，然后构建参数模型并进行安全性评估，减少调优过程中系统故障；3. 冷热数据感知技术，根据数据的访问频率、最近访问时间和重要性，感知各存储节点上的数据温度，将数据划分为不同的冷热等级，实现冷热数据的高效管理和优化访问；4. 智能缓存技术，包括基于工作负载特征和访问模式的智能预取算法、多级缓存体系结构中不同级别之间的数据迁移和准入策略、以及多节点层次化和平行化动态变化环境下的智能缓存替换策略。图1.2.2-1面向新型计算模式的 AI 大脑使能慢盘故障处理测试使用Cosbench工具启动负载，负载模型不做限制，用IOSTAT记录Ceph存储系统中osd所在三个节点的磁盘信息。随机选取一个NVMe利用nvme-hijac进行慢盘故障注入，然后对收集的IOSTAT信息用异常检测模块进行初步慢盘设备的识别，同时将识别的慢盘ID等信息传送给存储AI大脑，AI大脑依据观测信息将输出相关指令，用ceph指令观察相应osd状态可以看到对应操作的执行效果，如相应OSD被down掉。TC- P2负载预测准确率测试使用下载脚本命令，获取开源云原生负载，包括：微软Azure Function trace和华为Huawei Public Cloud trace；运行负载预测模型训练脚本，获取负载数据预测结果并将其保存至本地文件中；运行测试脚本，根据本地文件中存储的预测结果，统计微软Azure Function trace和华为Huawei Public Cloud trace两个负载的请求并发度和请求到达率。这两项负载特征指标预测结果均高于90%。TC- P3冷热数据识别准确率测试使用YCSB工具，参数设置：数据分布采用正态分布，单线程运行，测试记录条数为3000，操作数为9000；记录运行后日志中输出的准确率；准确率不低于70%。TC- P4IO 请求命中率检测使用5个节点（1个存储AI大脑节点，1存储节点，3个客户端节点）构建分布式环境，测试分布式缓存算法的IO请求命中率。使用开源工作负载进行测试，包括：webmail-all、twitter-042、twitter-049和twitter-020。设置缓存空间为工作集大小（唯一对象数）的40%，运行测试脚本，各个客户端每0.5s向存储AI大脑汇报该时间段内处理的IO请求数和IO请求命中数。当IO请求命中率稳定后，命中率不低于90%。TC- P5混合 I/O 负载动态选路硬件环境使用2物理节点，每个节点各带有一个 NVMe SSD 和一个 SATA SSD，其中一个节点作为负载压力测试的节点。压力测试程序使用自研工具，根据 JSON 文件配置的负载信息，生成 object 对象数据，使用追加写逻辑分配盘LBA地址，然后写入到盘上。混合 I/O 负载动态选路是由决策树算法，判断负载的优先级、大小、是否为热点数据，来选择要写入哪个盘。测试中具体的参数设置：运行五个负载。第一个负载 IO 大小为 4K，线程数为 1，不带热点，普通优先级；第二个负载 IO 大小为 128K，线程数为 1，不带热点，普通优先级；第三个负载 IO 大小为 4K，线程数为 1，带热点，普通优先级；第四个负载 IO 大小为 128K，线程数为 1，带热点，普通优先级；第五个负载 IO 大小为 4K，线程数为 1，不带热点，高优先级。五个负载一起运行，均运行60s。TC- P6存储 AI 大脑使能系统性能自动调优技术测试启动 AI 大脑后，运用 Rados bench 工具模拟系统负载并获取系统性能值。AI 大脑检测到系统负载情况，并在任务完成后获取到此次任务的 IOPS 值。由于IOPS 值处于低水平，AI 大脑的调参指令被触发。随即，AI大脑发出命令启动调参程序，在调参指令执行完毕后，再次运用 Rados bench工具对系统进行相同负载测试并得到测试结果，结果显示性能相较于调参前有显著提升。通过查看 ceph 的日志发现系统参数已被修改，AI大脑的命令执行成功。 测试结果测试场景TC-P1：存储 AI
