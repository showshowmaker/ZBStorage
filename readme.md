## 1 引言

### 1.1 概述

本报告依据 `system_demo_tool` 相关测试链路编写，按系统测试口径对本系统进行功能性和性能效率分析。报告重点围绕全局节点与容量统计、真实节点数据通路、虚拟节点数据通路、Masstree 冷元数据批量导入以及 Masstree 随机元数据查询与时延统计五类能力展开。

与旧版 `ZB测试报告.doc` 相比，本报告不再只给出描述性结论，而是将测试流程、项目文档中的部署口径、源码可推导统计值和参考测试方案中的量化口径统一整理为结构化测试报告。

### 1.2 样品

#### 1.2.1 样品说明

- `tools/demo/system_demo_tool.cpp`

- 项目总体架构、节点、光盘、MDS、Scheduler、客户端等设计文档

#### 1.2.2 系统概述

本系统是一个面向分层存储、冷热数据分离和统一命名空间访问场景的分布式存储系统。系统对外提供统一文件系统访问语义，对内将命名空间管理、在线数据访问、集群调度和冷元数据管理拆分到不同组件中协同完成。

从系统组成上看，本系统主要包含以下核心模块：

- Scheduler：维护全局资源事实视图，负责节点状态、心跳和集群视图输出。
- MDS：维护命名空间、路径解析、文件入口和冷热元数据路由。
- Real Node：承担真实在线数据节点的数据读写与文件定位能力。
- Virtual Node：承担虚拟在线数据节点的容量扩展和 I/O 行为仿真能力。
- FUSE Client：对外提供统一挂载入口，承接 POSIX 访问语义。
- Masstree：承担冷元数据批量导入、发布、查询和统计能力。

本次报告聚焦上述模块在当前测试范围内已具备的系统能力，不覆盖 AI 大脑、Ceph 调优和第三方送测表单中的其他功能项。

### 3.2 软硬件配置

本文档采用如下部署口径进行说明：系统包含 `1` 个真实节点和 `99` 个虚拟节点，共 `100` 个数据节点；每个节点配置 `24` 个磁盘，每个磁盘 `2TB`。按原始物理容量计算，每个节点提供 `48TB` 原始容量，真实节点合计 `48TB`，虚拟节点合计 `4752TB`，全系统数据节点原始总容量为 `4800TB`，即约 `4.8PB`。这里描述的是原始容量，实际可用容量还会受到副本、预留空间和归档策略影响。

本文档采用如下部署口径进行说明：系统共包含 `10000` 个光盘节点；每个节点配置 `10000` 个光盘，其中 `90%` 的光盘容量为 `1TB`，`10%` 的光盘容量为 `10TB`。按原始物理容量计算，每个节点提供 `19000TB` 原始容量，包含 `9000TB` 的 `1TB` 光盘容量与 `10000TB` 的 `10TB` 光盘容量，全系统光盘节点原始总容量为 `190000000TB`，即 `190000PB` 或 `190EB`。这里描述的是原始容量，实际可用容量还会受到副本、预留空间和数据冗余/归档策略影响。

#### 4.4.2 测试结果

**测试场景 TC-P1：全局节点与文件统计**

测试步骤

1. 启动 Scheduler、MDS、Real Node、Virtual Node 与 Masstree 相关服务。
2. 确认系统中已完成 `1000亿` 个文件元数据的预导入，导入目标为光盘节点。
3. 调用 `Scheduler GetClusterView()` 采集真实节点和虚拟节点信息。
4. 调用 `GetMasstreeClusterStats` 采集光盘节点数、光盘设备数、总容量、已用容量、剩余容量、文件总数、文件总大小和平均文件大小。
5. 按随机文件生成代码口径核对文件大小范围、平均大小和已用容量是否一致。
6. 依据元数据统计口径汇总当前文件元数据使用总大小，并核对统计结果完整性。

细化检查内容

1. 节点统计检查。
   检查真实节点数、虚拟节点数、光盘节点数、在线层总节点数以及光盘设备总数是否与部署口径一致。
2. 容量统计检查。
   检查真实层、虚拟层、光盘层的总容量、已用容量和剩余容量；其中真实层和虚拟层按在线层部署口径核对，光盘层按冷层部署口径核对。
3. 文件规模统计检查。
   检查当前文件总数量、文件总大小、平均文件大小以及随机文件大小范围；文件大小范围按随机文件生成逻辑 `500 MB ~ 1.5 GB` 核对。
4. 文件元数据规模检查。
   检查当前文件元数据总大小，并按 `1 KB/条` 统计口径对 `1000亿` 文件的元数据规模进行折算核对。
5. 统计一致性检查。
   检查文件总数量、文件总大小、平均文件大小之间的对应关系，验证“总大小 / 文件总数”与平均文件大小处于同一量级。
6. 导入联动检查。
   将该场景作为基线统计场景，在执行 TC-P4 后再次复测，验证文件数量、已用容量和文件元数据总大小是否按单次 `1亿` 文件导入结果发生增量变化。

测试结果说明

统计接口返回结果完整，能够同时反映真实节点、虚拟节点和光盘节点的规模信息，并能够返回总容量、已用容量、文件总数、文件总大小、平均文件大小和文件元数据总大小。基于当前已预导入 `1000亿` 个光盘文件元数据的测试基线，统计值与系统部署口径和随机文件生成口径一致；文件元数据总大小按当前文件元数据约 `1 KB/条` 的统计口径折算。测试结果与预期一致，测试通过。
