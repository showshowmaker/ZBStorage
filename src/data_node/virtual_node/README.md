# virtual_node module

## Role
`virtual_node` is a simulated data node. It exposes the same object I/O RPC interface as `real_node` but uses latency/bandwidth simulation instead of real disk persistence.

## Core responsibilities
- Simulate object write/read latency by configured throughput and base delay.
- Return object read data generated by the simulator path.
- Report configured virtual disks and capacities.
- Participate in scheduler heartbeat as a normal data node.

## Run
```bash
./build/virtual_node_server --config=config/virtual_node.conf.example --port=29080
```
